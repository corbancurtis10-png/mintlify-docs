EZAnim exposes two event callbacks that let your game logic react to animation state changes: `onMarker` fires when a named keyframe marker is reached, and `onStopped` fires when an animation track stops playing.

## Animation Marker Events

Roblox lets you embed named **keyframe markers** in animations using the Animation Editor. `onMarker` connects a callback to fire each time the animation passes that marker.

Both `onMarker` and `onStopped` use `getAnim` internally, so the track must already exist before you call them. Call `play` first, then set up your event connections:

```lua
-- Play the animation first so the track exists
anim:play(character, "attack", "Action", false)

-- Then connect the marker callback
local connection = anim:onMarker(character, "attack", "HitFrame", function()
    print("Hit frame reached â€” deal damage!")
    dealDamage(target)
end)
```

`onMarker` returns an `RBXScriptConnection?` â€” it returns `nil` if the track was not found. Always store and check the return value if you need to disconnect later:

```lua
local connection = anim:onMarker(character, "attack", "HitFrame", function()
    -- ...
end)

if connection then
    -- Disconnect when no longer needed
    connection:Disconnect()
end
```

<Warning>
  `onMarker` uses `getAnim` internally and returns `nil` if the track does not exist yet. Always call `play` before wiring up `onMarker`, or ensure the track was loaded by a previous `play` call.
</Warning>

## Stopped Event

`onStopped` connects a callback that fires once the animation track fully stops. Like `onMarker`, call `play` first so the track exists, then call `onStopped`:

```lua
-- Play first so the track is loaded
anim:play(character, "death", "Action", false)

-- Then connect the stopped callback
local conn = anim:onStopped(character, "death", function()
    print("Death animation complete â€” respawn player")
    respawnPlayer(player)
end)
-- conn is RBXScriptConnection? â€” nil if track was not found
```

## Chaining Animations

A common pattern is to chain a sequence of animations using `onStopped`:

```lua
-- Play intro, then loop idle
anim:play(character, "spawn_intro", "Action", false)

anim:onStopped(character, "spawn_intro", function()
    anim:play(character, "idle", "Idle", true)
end)
```

## Combining Markers and Stopped

```lua
-- Play a combo attack first, then connect events
anim:play(character, "combo", "Action", false)

-- Deal damage at the hit frame
anim:onMarker(character, "combo", "Hit", function()
    applyDamage(target, 25)
end)

-- Reset combo state when animation ends
anim:onStopped(character, "combo", function()
    resetComboState(character)
end)
```

<Tip>
  Both `onMarker` and `onStopped` return `RBXScriptConnection?` â€” `nil` if the track was not found. Check the return value when you need to disconnect, and always ensure `play` has been called first for that model and animation key.
</Tip>
