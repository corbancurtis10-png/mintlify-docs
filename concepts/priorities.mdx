When two or more animations play at the same time on a character, Roblox uses the animation's **priority** to determine which one takes precedence. EZAnim accepts a `type` string on every `play` call and attempts to map it to a `Roblox Enum.AnimationPriority` value.

## How Priority Lookup Works

Internally, EZAnim lowercases the `type` argument before performing the lookup:

```lua
type = string.lower(type or "movement")
track.Priority = PriorityMap[type] or Enum.AnimationPriority.Movement
```

The internal `PriorityMap` has **capitalized** keys (`"Idle"`, `"Movement"`, `"Action"`, etc.). Because the input is lowercased before lookup, the lowercased string (e.g. `"idle"`) never matches a capitalized key (e.g. `"Idle"`), so the lookup always returns `nil` and the priority always falls back to `Enum.AnimationPriority.Movement`.

<Warning>
  Due to the mismatch between the lowercased lookup key and the capitalized map keys, **the `type` parameter has no effect on priority in practice**. Every animation is played at `Enum.AnimationPriority.Movement` regardless of the value you pass.
</Warning>

## Setting Priority

Even though the `type` parameter does not currently affect priority, you should still pass it correctly so your code is forward-compatible and self-documenting:

```lua
-- All of these will play at Enum.AnimationPriority.Movement in practice
anim:play(character, "idle", "Idle", true)
anim:play(character, "run", "Movement", true)
anim:play(character, "attack", "Action", false)
```

## Default Priority

If you omit `type` or pass an unrecognized string, EZAnim explicitly defaults to `Enum.AnimationPriority.Movement` — the same value used for all inputs:

```lua
-- These all result in Enum.AnimationPriority.Movement
anim:play(character, "run")              -- type omitted
anim:play(character, "run", "Action")    -- type recognized but lowercased before lookup
anim:play(character, "run", "unknown")   -- type unrecognized
```

## Intended Priority Levels

The following table shows the intended mapping as defined in the source. These values are not reachable through the `type` parameter today, but represent the design intent:

| Intended `type` string | Roblox Enum | Typical Use |
|------------------------|-------------|-------------|
| `"Idle"` | `Enum.AnimationPriority.Idle` | Standing still, breathing cycles |
| `"Movement"` | `Enum.AnimationPriority.Movement` | Walking, running, swimming |
| `"Action"` | `Enum.AnimationPriority.Action` | Attacking, interacting, emoting |
| `"Action2"` | `Enum.AnimationPriority.Action2` | Secondary action layer |
| `"Action3"` | `Enum.AnimationPriority.Action3` | Tertiary action layer |
| `"Action4"` | `Enum.AnimationPriority.Action4` | Highest priority action |

Higher priority overrides lower priority. `Action4` > `Action3` > `Action2` > `Action` > `Movement` > `Idle`.
